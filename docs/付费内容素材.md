## 一、定价策略与报价模板

### 1.1 定价公式

```
时薪 = (目标月收入 + 固定成本) ÷ 可计费小时数 × 1.3
```

**示例计算**：
- 目标月收入：20,000 元
- 固定成本（社保、设备、房租分摊）：3,000 元
- 每周可计费工时：25 小时 × 4.3 周 = 107 小时
- 时薪 = (20000 + 3000) ÷ 107 × 1.3 ≈ **280 元/小时**

### 1.2 报价模板

```
【项目报价单】

项目名称：_______________
客户名称：_______________
报价日期：_______________

一、需求概述
[简要描述客户需求，3-5句话]

二、工作范围
✅ 包含：
- 功能点 1
- 功能点 2
- 功能点 3

❌ 不包含：
- UI 设计（需客户提供设计稿）
- 服务器运维
- 后期功能迭代

三、报价明细
| 模块 | 工时估算 | 单价 | 小计 |
|------|---------|------|------|
| 前端开发 | 40h | ¥280 | ¥11,200 |
| 后端开发 | 30h | ¥280 | ¥8,400 |
| 联调测试 | 10h | ¥280 | ¥2,800 |

总计：¥22,400

四、付款方式
30% 预付 → 40% 中期验收 → 30% 交付前结清

五、交付周期
预计 15 个工作日，自收到预付款后开始计算

六、有效期
本报价有效期为 7 天
```

### 1.3 定价策略技巧

1. **不报单一数字**：给出范围，如「1.5-2.5万」
2. **先问预算**：「您这个项目的预算范围大概是？」
3. **拆分报价**：按模块/阶段报价，增加透明度
4. **预留谈判空间**：报价时加 20% 缓冲

## 二、合同要点

### 2.1 必须包含的条款

1. **项目范围**：明确做什么，不做什么
2. **付款条款**：金额、时间节点、支付方式
3. **知识产权**：源码归属、商用授权
4. **变更管理**：需求变更如何处理、收费标准
5. **验收标准**：什么情况算完成
6. **保密条款**：双方保密义务
7. **违约责任**：双方违约的处理方式

### 2.2 付款节点建议

| 阶段 | 比例 | 触发条件 |
|------|------|---------|
| 预付款 | 30% | 合同签订后 3 个工作日 |
| 中期款 | 40% | 核心功能演示通过 |
| 尾款 | 30% | 源码交付前 |

### 2.3 风险提示

⚠️ **高风险信号**：
- 要求先做再付款
- 无法提供公司信息
- 催得很急但不愿签合同
- 反复修改需求但不加钱

## 三、客户沟通话术

### 3.1 初次沟通

```
您好！感谢您的咨询。

为了更准确地给您报价，我需要了解几个问题：
1. 这个项目的核心功能是什么？
2. 您期望的上线时间是？
3. 有没有参考的产品或设计稿？
4. 您这边的预算范围大概是？

期待您的回复！
```

### 3.2 议价应对

**客户说「太贵了」**：
```
理解您的顾虑。这个价格是基于 [X] 小时的工作量估算的。

我们可以这样调整：
1. 缩减部分非核心功能，降低工作量
2. 分期开发，先上线核心版本
3. 如果您能提供设计稿，可以省去设计费用

您看哪种方式更适合？
```

### 3.3 需求变更

```
收到您的新需求。

这部分内容不在原合同范围内，需要额外工时：
- 新增功能：约 8 小时
- 费用增加：¥2,240

如果确认添加，我会更新报价单。
原计划交付时间会顺延 2 个工作日。

请确认是否继续？
```

### 3.4 催款话术

```
您好！

项目已完成中期验收（详见附件截图）。
根据合同约定，请于 3 个工作日内支付中期款 ¥____。

收款信息：
- 账户：
- 开户行：

收到款项后，我会继续后续开发。
如有疑问，随时沟通。
```

## 四、税务与财务

### 4.1 收入类型

| 身份 | 税种 | 税率参考 |
|------|------|---------|
| 个人 | 劳务报酬所得 | 20-40%（可抵扣） |
| 个体户 | 经营所得 | 5-35%（核定征收更低） |
| 公司 | 企业所得税+个税 | 综合约 10-20% |

### 4.2 省税建议

1. **注册个体户**：部分地区有核定征收，综合税负可低至 3-5%
2. **合理抵扣**：电脑、软件、培训等费用可抵扣
3. **开票选择**：小规模纳税人每月 10 万以下免增值税
4. **咨询专业人士**：金额较大时建议咨询会计

### 4.3 财务管理

- 单独开设「接单收入」账户
- 每月记录收支明细
- 预留 20-30% 作为税金
- 设立应急基金（3-6个月生活费）

## 五、接单避坑指南

### 5.1 🚩 危险信号

| 信号 | 风险等级 | 建议 |
|------|---------|------|
| 先做后付 | ⚠️⚠️⚠️ | 坚决拒绝 |
| 催得很急 | ⚠️⚠️ | 加收急单费 |
| 需求模糊 | ⚠️⚠️ | 要求书面确认 |
| 压价严重 | ⚠️ | 评估是否值得 |
| 不愿签合同 | ⚠️⚠️⚠️ | 直接放弃 |

### 5.2 自我保护

1. **保留证据**：所有沟通记录、验收确认
2. **分阶段交付**：不要一次交付全部源码
3. **演示不给源码**：中期演示在自己服务器
4. **设定有效期**：报价超过 7 天需重新评估
